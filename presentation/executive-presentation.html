Executive Summary

The InsureCo Claims Submission Platform is a modern, cloud-native solution designed to streamline the insurance claim submission and processing workflow.

Key Highlights

  • Microservices Architecture: Scalable, maintainable, and resilient
  • Event-Driven Processing: Asynchronous claim adjudication with Kafka
  • Multi-Language Stack: Best tool for each job (Node.js, Python, Go, Java)
  • Cloud-Ready: Containerized with Docker, deployable to any cloud platform
  • Security First: OAuth2/JWT authentication, TLS encryption, rate limiting

Architecture Overview

🌐 Presentation Layer

Claims Portal: React 18 + TypeScript SPA with multi-step form validation

CDN: CloudFront/S3 for static asset delivery with edge caching

Auth Service: AWS Cognito/Auth0 for OAuth2 authentication

🔀 API Gateway Layer

Kong/AWS API Gateway: TLS termination, JWT validation, rate limiting (100 req/min)

Single ingress point for all backend services with request tracing

⚙️ Microservices Layer

Claims Service (NestJS): Core orchestrator for claim submission

Document Service (FastAPI): File handling with virus scanning

Notification Service (Go): Email/SMS dispatch via SES/Twilio

💾 Data & Messaging Layer

PostgreSQL (RDS Multi-AZ): Primary data store with row-level security

S3: Document storage with lifecycle policies

Kafka (MSK): Event streaming backbone (3 partitions, RF=3)

🔄 Processing Layer

Claims Processor (Java/Spring): Async adjudication worker with auto-triage

Consumes events, applies business rules, assigns adjusters

End-to-End Claim Submission Flow

1
Policyholder Submits Claim

Multi-step form with client-side validation → POST /api/claims with JWT Bearer token

2
API Gateway Validates

TLS 1.3 termination → JWT signature verification → Rate limit check → Route to Claims Service

3
Claims Service Processes

Validate payload → Coverage check → Duplicate detection → BEGIN TRANSACTION → INSERT claim + documents

4
Kafka Event Published

COMMIT transaction → Produce to claims.submitted topic → Return 201 with claimId (~310ms)

5
Async Processing

Claims Processor consumes event → Auto-triage (< $5K = auto-approve) → Update status → Assign adjuster

6
Notifications Sent

Notification Service consumes status update → Send email via SES → Send SMS via Twilio (~1.2s total async)

Security Features

Authentication & Authorization

  • OAuth 2.0 / OIDC: Industry-standard authentication with AWS Cognito/Auth0
  • JWT RS256: Asymmetric token signing with JWKS endpoint verification
  • Role-Based Access: Policyholder, Adjuster, and Admin roles with granular permissions
  • MFA Enforced: Multi-factor authentication for sensitive operations

Data Protection

  • TLS 1.3: End-to-end encryption for all API communications
  • SSE-S3: Server-side encryption at rest for document storage
  • Row-Level Security: PostgreSQL RLS restricts data access by user role
  • Virus Scanning: ClamAV integration for uploaded documents

API Security

  • Rate Limiting: 100 requests per minute per IP address
  • Input Validation: Zod schema validation on client and server
  • SQL Injection Prevention: Parameterized queries throughout
  • CORS Configuration: Strict origin policies for API access
